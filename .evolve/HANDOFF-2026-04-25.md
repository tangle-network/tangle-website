**What the patch does** (single behavior change):

`app/api/chat/route.ts` `checkCredits()` now treats an active non-free
`Subscription` row as unlimited entitlement (`return Number.POSITIVE_INFINITY`),
short-circuiting the free-tier daily cap that would otherwise kick in for
any subscriber whose metered credit balance happens to be zero.

```ts
// after the platform-balance + local-balance checks both return 0:
const subOr = teamId ? [{ userId }, { teamId }] : [{ userId }]
const activeSub = await prisma.subscription.findFirst({
  where: { OR: subOr, status: 'active', NOT: { plan: 'free' } },
  select: { id: true },
}).catch(() => null)
if (activeSub) return Number.POSITIVE_INFINITY
return 0
```

The 60 req/min per-API-key rate limit (`route.ts:1300`) is unchanged — this
patch only kills the **daily** cap for paid users, not the burst limiter.

### If still rate-limited after deploy lands

The patch is correct but only helps users whose `Subscription` row exists
in the router's prod DB with `status: 'active'` AND `plan != 'free'`. If
the Stripe webhook never populated that row, you'll still hit the 20/day
cap.

**Diagnose** (requires SSH to drew-gtr-pro since the postgres is on a
docker network not exposed publicly):

```bash
ssh drew@drew-gtr-pro
docker exec -it $(docker ps -q -f name=tangle-router-postgres) \
  psql -U tangle_router -d tangle_router \
  -c "SELECT id, plan, status, \"stripeSubscriptionId\", \"currentPeriodEnd\" FROM \"Subscription\" WHERE \"userId\" = 'YOUR_USER_ID';"
```

Three possible outcomes:

| Result | Meaning | Fix |
|---|---|---|
| Row exists, `status='active'`, `plan` in `{'pro','enterprise'}` | Subscription is honored — patch should work post-deploy | (no fix; check deploy actually landed) |
| Row exists, `status='canceled'` or `plan='free'` | Stripe state diverged from DB | Re-trigger Stripe webhook OR insert a manual override |
| No row | Webhook never fired or never matched user | Insert manually (template below) |

**Manual seed template** (run on the router prod DB; adjust IDs):

```sql
INSERT INTO "Subscription" (
```
