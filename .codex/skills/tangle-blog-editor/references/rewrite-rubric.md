# Tangle Blog Rewrite Rubric

Score each dimension 1-5.

| Dimension | 1 | 5 |
|---|---|---|
| Reader pressure | Generic explainer | Specific builder/operator/agent problem |
| Tangle proof | Name-drops Tangle | Shows concrete Tangle mechanism or endpoint |
| Technical precision | Broad AI wording | Exact surfaces, gates, commands, risks |
| SEO/AEO fit | Query unclear | Query answered early and FAQ supports it |
| Style | Research-note template | Builder voice with varied structure |
| Conversion | No next action | Specific install, curl, manifest, CTA, or decision |
| Evidence | Unsupported claims | Primary sources, internal links, proof blocks |

## Rewrite Pass Order

1. Frontmatter: title, slug, summary, tags, series order.
2. Opening: first 150 words must answer why this post exists.
3. Structure: reduce repeated scaffold headings; merge thin sections.
4. Proof: add commands, manifests, links, images, or code where the claim needs evidence.
5. Retrieval: add answer capsule, comparison table, and FAQ where useful.
6. Tangle: move specific product mechanism earlier.
7. Ending: replace summary with decision or readiness test.

## Existing Series Diagnosis

The current self-improving stack posts are strong on systems thinking and weak on origin pressure. They often begin from abstraction and then explain the map. Upgrade them by starting from the workflow failure that forced the abstraction:

- Prompt optimization: agent follows better wording but cannot gain missing runtime capability.
- Skill optimization: a procedure persists, triggers too broadly, and becomes behavioral debt.
- Runtime topology: "parallelize" does nothing unless the runtime exposes real fanout.
- Multi-agent coordination: five named roles can share one correlated blind spot.
- Test-time compute: guided reasoning must beat random sampling at equal budget.
- Evaluation gates: the gate defines what persists.
- Trace systems: scores lose mechanism.
- Harness evolution: the current machine cannot express the needed move.
- Post-training: weight changes move the rollback and governance boundary.
- Memory: retrieval is an intervention, not proof of learning.
- Governance: self-improvement needs a control plane outside the optimized surface.
